noaqh-devコマンドを以下の手順でリリースしなさい。

## 手順
1. どのようなリリースにするのかを、未pushコミット、未コミット変更をもとに判断し、ユーザーにテンプレート通りversion等の情報を確認しなさい。
2. ユーザーがリリース情報を承認した場合は処理を続けなさい。
3. package.jsonのversionを更新。
4. tagを作成し、releaseを作成しなさい。また、tagとreleaseの名前はv0.1.1のようにバージョンを含めるようにしなさい。
5. releaseにはリリースノートも作成し、pushしなさい。

## 確認テンプレート
```

以下の内容でリリースします。本当にリリースしますか？ (y/n)

# リリース情報
## バージョン
- 現在のバージョン: v{{ current_version }}
- リリースするバージョン: v{{ release_version }}

## リリース内容
- リリース内容: {{ release_content }}

## 対象コミット
- {{ 対象コミットの1行目 }}
- {{ 対象コミットの1行目 }}
- {{ 未コミットのものがあればコミット予定のものも含める }}
...


```
