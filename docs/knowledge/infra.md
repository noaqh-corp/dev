# インフラガイド

## 選定基準
- 並列処理が必要な場合を除きサーバーレスアーキテクチャは採用しない。
- インフラコストを多少削減するために複雑なアーキテクチャを採用しない。
- AWS、GCP、Azureは顧客の強い要望があった場合、もしくは要件上必須の場合のみ採用
- 基本的にはRenderを採用
    - Web Service + DB のシンプルな構成を推奨
- オブジェクトストレージが必要な場合はCloudflare R2を採用

## Infra as Code
基本的にはPulumiを採用

https://www.pulumi.com/
